# 미션 : GenAI 기초 1: LLM 기반 업무 자동화

## 1. 미션 개요 및 최종 결과물
본 미션은 LLM을 활용하여 실제 업무를 자동화할 수 있는 '프롬프트 패키지'를 완성하는 과정입니다. 최종적으로 아래 3가지 산출물을 제출해야 합니다.

* **LLM 모델 비교·선정 보고서 (문서 1개):** 동일한 과업 프롬프트를 3종 이상의 모델에 입력하고, 결과를 비교한 표와 최종 선정 결론(근거 포함)을 담은 문서.
* **시스템 설계 문서 (문서 1개):** 타겟 사용자, 문제 정의, 페르소나, 시스템/유저 프롬프트, Few-shot 예시, 환각 검증 전략, 프롬프트 개선 이력(v1→v2) 및 최종 프롬프트 전문을 담은 문서.
* **실행 로그 (대화 로그 1개):** 선택한 시나리오로 10턴 이상 실제 대화를 수행한 로그. 문제 발생 지점(환각/문맥 끊김)과 수정 결과 요약이 포함되어야 합니다.

---

## 2. 과제 목표
미션을 완료한 학습자는 다음을 명확히 설명하고 적용할 수 있어야 합니다.

* LLM의 "확률적 생성" 원리와 환각(Hallucination) 발생 원인 설명.
* Zero-shot, Few-shot, 단계적 추론 유도(Reasoning Prompt) 기법의 차이 구분 및 적재적소 적용.
* 단순한 "느낌"이 아닌 명확한 평가 축과 근거를 바탕으로 3종 이상의 LLM 비교 및 선택.
* 페르소나와 시스템 프롬프트가 결과물의 품질(일관성/안전성/형식 준수)에 미치는 영향 파악.
* 생성 결과를 테스트로 검증하고(사실 오류 위주), 프롬프트를 반복 개선하는 과정의 재현.

---

## 3. 기능 요구 사항
과제 수행 시 반드시 만족해야 하는 8가지 핵심 요구사항입니다.

1.  **모델 비교 및 선정 (최소 3종):**
    * GPT-4o, Claude 3.5 Sonnet, Gemini 1.5 Pro 등 3개 이상의 LLM 선정 및 동일 과업 수행.
    * 보고서 필수 항목: 비교 대상 모델명, 사용 환경(웹/앱/API), 평가 축(최소 4개), 모델별 점수(1~5점) 및 근거, 최종 선정 결론(선정 이유 3줄 이상).
2.  **업무 자동화 과업 1개 선택 및 문제 정의:**
    * 메일 초안 작성, 회의록 요약, 보고서 개요 생성 등 본인 업무에 준하는 과업 정의.
    * 명시 사항: 타겟 사용자, 입력 데이터 형태, 출력 규격.
    * 재사용 가능한 "입력 템플릿" 1개 이상 제시.
3.  **페르소나 정의 및 시스템 프롬프트 설계:**
    * 페르소나 구체화 (이름/역할/전문분야/말투/금지사항/우선순위).
    * 시스템 프롬프트 필수 포함: 역할과 목표, 출력 형식 규칙, 안전장치(모르면 모른다고 하기 등), 사실/수치 처리 규칙.
4.  **Few-shot 예시 (최소 3개) 포함:**
    * "좋은 입력 → 좋은 출력" 예시 최소 3개 구성.
    * 이 중 1개는 "모호한 입력" 케이스로 구성하여 AI가 되물어야 하는 지점 명시.
5.  **단계적 추론 유도 적용 및 전/후 비교:**
    * 동일 입력에 대해 `v1(일반 지시)`과 `v2(단계적 접근 유도)` 프롬프트 결과 비교.
    * 불필요한 추론 과정 노출 방지를 위해 "최종 답변은 요약/근거 중심" 규칙 적용.
6.  **환각(Hallucination) 검증:**
    * 사실 기반 검증 질문 5개 이상 준비 (기대 정답/허용 오차/참고자료 명시).
    * **Pass 기준:** 정답 및 근거 제시 또는 "모름/확인 필요" 명시 후 확인 절차 제안.
    * **Fail 기준:** 틀린 사실 단정적 발언 또는 불명확한 전제에서 임의로 사실 생성.
7.  **10턴 이상 대화 로그 + 문맥 유지 검증:**
    * 선택 시나리오로 10턴 이상 대화 진행. 중간에 조건 변경(톤/대상/정책 변경)이나 추가 정보 뒤늦게 제공하는 상황 반드시 포함.
    * **문맥 유지 Pass 기준:** 확정된 요구사항 10턴 내내 유지, 조건 변경 시 변경된 부분만 반영하고 나머지는 유지.
8.  **제출물 최소 목차:** 형식은 자유이나 모델 비교·선정 보고서, 시스템 설계 문서, 실행 로그의 필수 항목들을 모두 포함해야 함.

## 4. 제약 사항
과제 수행 및 제출 시 반드시 지켜야 할 제약 조건입니다.

* **모델/환경 기록(재현성):**
    * 제출물에 다음 항목 필수 포함: 모델명, 요금제(무료/유료), 사용 채널(웹/앱/API), 사용 날짜, 주요 설정(temperature/top_p/max_tokens 등).
    * 무료 버전 사용 시 "무료라서 제한된 점"을 1줄로 명시.
* **안전/윤리:**
    * 폭력적/선정적 콘텐츠 생성 시도 절대 금지.
    * 개인정보(실명, 전화번호, 주소, 계정 정보 등) 프롬프트 및 로그에 직접 포함 금지.
    * 사실/정책/수치가 포함되는 답변은 "근거/확인 필요" 규칙을 반드시 따를 것.
* **제출물 편집:**
    * 가독성을 위해 대화 로그를 그대로 덤프하지 말고 섹션/번호/요약을 추가하여 정리.
    * 단, 재현성 확보를 위해 "원본 로그(전문)"는 별도 파일로 남길 것.

## 5. 결과 예시 (참고용)
각 과제 항목에 대한 작성 방향을 보여주는 참고 예시입니다. (실제 구성 및 문구는 자유롭게 작성)

* **모델 비교 보고서:**
    * *비교 축(예시):* 정확성, 한국어 자연스러움, 긴 문맥(10턴) 유지, 형식 준수, 속도/안정성, 비용, 도구 연동 적합성.
    * *결론(예시):* "'회의록 요약' 과업은 긴 문맥 유지와 형식 준수가 중요하므로, 해당 축 점수가 높은 모델을 채택."
* **업무 과업 입력 템플릿:**
    ```text
    [업무 과업] 회의록 요약
    [원문] (회의 메모/녹취 요약 텍스트)
    [원하는 결과] 결정사항 / Action Items / 리스크 / 후속 일정
    [톤] 간결, 사내 공유용
    [금지] 추측성 표현 금지, 인물 실명 대신 역할명 사용
    [확인 질문 규칙] 정보가 부족하면 최대 3개까지 질문 후 요약 시작
    ```
* **페르소나 및 시스템 프롬프트 요약:**
    * *페르소나:* "프로젝트 매니저 출신 업무 자동화 코치"
    * *원칙:* (1) 사실/수치/정책은 확인 가능해야 함 (2) 부족하면 질문 (3) 최종 출력은 템플릿 준수
* **Few-shot 예시 (모호한 입력 케이스):**
    * *입력:* "회의 메모가 너무 길어. 결정사항과 할 일만 뽑아줘. 대신 확실한 내용만."
    * *출력:* "- 결정사항: ... 
 - Action Items: 담당/기한/작업 ... 
 - 확인 필요: ... (근거 부족 항목)"
* **단계적 추론 유도 프롬프트 전/후:**
    * *v1(간단):* "아래 회의 내용을 요약해줘."
    * *v2(개선):* "1) 먼저 목표 출력 항목(결정/할일/리스크) 선언 -> 2) 정보 부족 시 최대 3개 질문 -> 3) 마지막에 템플릿 출력 -> 4) 최종 답변에 핵심 근거 3개 덧붙이기"
* **환각 검증 기록:**
    * *질문:* "(업무 규정) 출장비 정산 규정의 예외 조건이 뭐야?"
    * *Pass:* "규정 문서가 없으면 확답 불가. 사내 위키의 '예외' 섹션을 확인해야 함. 확인 후 요약 제공 가능."
    * *Fail:* "예외는 무조건 ... 입니다." (근거 없이 임의 답변)
* **10턴 대화 로그 흐름:**
    * 사용자 원문 제공 → AI의 누락 정보 확인 질문 2개 → 사용자 추가 정보 제공 → AI 템플릿 요약 출력 → (중략) → 사용자: "톤을 더 단호하게, 책임자를 역할명으로 바꿔줘" → AI: 변경 반영 + 기존 결정사항 유지.

## 6. 보너스 과제 (선택 사항)
본 과제 외에 추가적인 역량 강화를 위해 선택적으로 수행할 수 있습니다.

1.  **나만의 봇 배포 (재사용성 강화):** 커스텀 인스트럭션, GPTs, 또는 에이전트 기능을 활용해 최종 프롬프트를 '재사용 가능한 형태'로 배포해 보기.
2.  **멀티모달 확장 (업무 결과 시각화):** 생성된 결과(보고서, 요약 등)를 바탕으로 이미지 생성 AI를 활용해 1장 이상의 시각 자료 제작해 보기.
